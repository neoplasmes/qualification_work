# common
общие утилитарные функции, переиспользуемые во всём приложении (по большей части в implementation)

---

# core
### Бизнес-логика: (use case ориентированная)
__**commands:**__ POST/PUT и прочие запросы с большими body.<br/>
__**query:**__ GET и т.п.<br/>
### Адаптеры
__**ports:**__ интерфейсы для компонентов программы, реализующих непосредственную работу с данными, где _services_ - некие "утилиты", а _repositories_ - средства работы с данными, разделённые по т.н. бизнес-сущностям<br/>

---

# implementation
Реализация _services_ и _repositories_. А также реализация command-query ядра программы, которая по сути реализуется через REST api.

# infrastructure 
просто подключения к сторонним сервисам. _В перспективе и к **микро-сервисам**_
